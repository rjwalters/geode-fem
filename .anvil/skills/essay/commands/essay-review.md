# essay-review — Reviewer

**Role**: reviewer (single content critic + deterministic gate runner).
**Reads**: latest `<thread>.{N}/<thread>.md`, project `BRIEF.md` (+ resolved `voice:` docs), `<thread>/refs/` + shared `research/`, `rubric.md`, any consumer `.anvil/skills/essay/rubric.overrides.md` (additive only), prior `<thread>.{M}.review/` siblings (M < N) to track whether prior issues were addressed.
**Writes**: `<thread>.{N}.review/` with `verdict.md`, `scoring.md`, `comments.md`, `_summary.md`, `_gate.json`, `_meta.json`, `_progress.json` — plus the gate sidecars `<thread>.{N}.numeric/` and `<thread>.{N}.hyperlinks/` written by the gate CLIs themselves.

The review sibling is **read-only once written**. Revisions consume it; they never modify it.

## Outputs

```
<thread>.{N}.review/
  verdict.md       Advance / block + total /44 + critical-flag paragraphs + top revision priorities
  scoring.md       Per-dimension table: # | Dimension | Weight | Score | Justification
  comments.md      Line-level comments (severity blocker/major/minor/nit + scope preserve/expand/reduce)
  _summary.md      Machine-readable blocks: rubric block, voice_grounding (when author tier active), subject_voice_grounding (when subject tier active — #598), provenance_back_check (when corpus tier active — #611), gates echo, scope_distribution
  _gate.json       Deterministic pre-flight record (see step 3)
  _meta.json       { critic, role, started, finished, model, schema_version, scorecard_kind: "human-verdict",
                     rubric_id: "anvil-essay-v1", rubric_total: 44, advance_threshold: 35 }
  _progress.json   Phase state for the reviewer
```

**Atomicity** (issues #350, #376): written atomically via `anvil/lib/sidecar.py` — files staged under `.<thread>.{N}.review.tmp/`, atomically renamed on clean completion; stale staging from a prior interrupt of THIS critic removed by `cleanup_one_staging(<thread>.{N}.review)` at entry; the gate CLIs manage their own `.numeric/` / `.hyperlinks/` sidecars.

## Procedure

1. **Discover state, sweep, open sidecar**: find the highest `N` with `<thread>.{N}/<thread>.md` (slug-echo per #295); run `cleanup_one_staging(<thread>.{N}.review)`; if `<thread>.{N}.review/` exists, exit early (idempotent). Otherwise open `staged_sidecar(final_dir=<thread>.{N}.review, required_files=["verdict.md", "scoring.md", "comments.md", "_summary.md", "_gate.json", "_meta.json", "_progress.json"])` and write everything inside the staging dir. Initialize `_progress.json` and `_meta.json` with `scorecard_kind: "human-verdict"`, **`rubric_id: "anvil-essay-v1"`, `rubric_total: 44`, `advance_threshold: 35`** (per-review version stamping, issue #346). Load `<thread>.{N-1}.review/_meta.json` when present and cache `prior_rubric_id` for the `_summary.md` rubric block (omit on first iteration).

   **Non-Python-driver ordering (fail-open, manual fallback)** — issue #645: `staged_sidecar` is a Python context manager. A manual/agent session with **no orchestrating Python driver** cannot hold its `with` block open across the file writes below (it writes files with its own editing tool between discrete steps), so it MUST use the equivalent CLI shim rather than writing straight into the final `<thread>.{N}.review/` dir (which silently reopens the #350 partial-write defect this primitive exists to close). Two tiers, in preference order:

   1. **Primary — `python -m anvil.lib.sidecar` CLI shim** (the common case). In an installed consumer repo (anvil vendored under `.anvil/`, not on `sys.path`), prefix every invocation below with `uv run --project .anvil` (the `.anvil/pyproject.toml` + `uv sync --project .anvil` shipped by the installer since #230 make the module resolvable from the consumer root — the same shape the other `anvil.lib.*` critics already use); in the anvil source repo the bare `python -m anvil.lib.sidecar` form works as-is. This wraps the *exact same* `staged_sidecar` code, so the manifest check + single atomic `Path.rename` are enforced by code, not agent discipline:
      - `uv run --project .anvil python -m anvil.lib.sidecar stage <thread>.{N}.review` → prints the staging path (`.<thread>.{N}.review.tmp/`). (Refuses with a nonzero exit if `<thread>.{N}.review/` already exists — matching `staged_sidecar`'s `FileExistsError` refuse-to-overwrite guard.)
      - Write **all** required files (`verdict.md`, `scoring.md`, `comments.md`, `_summary.md`, `_gate.json`, `_meta.json`, `_progress.json`) into that printed staging path — never into the final `<thread>.{N}.review/` name.
      - `uv run --project .anvil python -m anvil.lib.sidecar commit <thread>.{N}.review --required verdict.md,scoring.md,comments.md,_summary.md,_gate.json,_meta.json,_progress.json` → verifies the manifest, then atomically renames staging → final. **Nonzero exit (1) leaves the staging dir in place with no partial final dir** if any required file is missing — the `SidecarIncompleteError` analog; fix the gap and re-`commit`.
      - The stale-staging sweep of step 1 has an exact CLI analog: `uv run --project .anvil python -m anvil.lib.sidecar cleanup <thread>.{N}.review` (the parallel-safe per-critic sweep, issue #376).
   2. **Last resort — manual `mv`-based staging** when even `python`/`uv` is unavailable. Reproduce the staging contract by hand: (a) at entry, sweep any leftover `rm -rf .<thread>.{N}.review.tmp/` (the `cleanup_one_staging` analog); (b) `mkdir .<thread>.{N}.review.tmp/` and write **every** required file into it — writing `_progress.json` **last**, so a mid-write interrupt is caught by the missing-manifest check rather than producing a final-named partial; (c) confirm the staging dir holds the full required set — use a count check (`[ "$(ls -1 .<staging-dir> | wc -l)" -eq <N> ]`) or an `ls`-based presence check rather than per-file `[ -f ]`, which can false-negative under a restricted-`stat` sandbox — **then** `mv .<thread>.{N}.review.tmp <thread>.{N}.review` as the **last** step (POSIX `mv` on a same-filesystem dir-to-dir rename is atomic, matching `Path.rename`). Do NOT create `<thread>.{N}.review/` before all files are staged. **Record the fallback durably** so a reader can tell atomicity was reproduced by hand rather than tool-verified: stamp `_meta.json` with `"atomicity_fallback": "manual-mv"` (e.g. `sidecar: staged_sidecar CLI unavailable (uv/python not on PATH); atomicity reproduced via manual mv this pass`). Absent this note the manual staging is indistinguishable from an unsafe direct write.

   The two tiers land a byte-identical on-disk result to the `staged_sidecar` context-manager path; they exist only to give a Python-less session a code-enforced (tier 1) or contract-faithful (tier 2) route to the same atomicity guarantee. When an orchestrating Python driver IS present, use `staged_sidecar` directly as documented above — the CLI shim is not needed.

2. **Read inputs**: the body, the matching BRIEF `documents:` entry (`target_length` when declared; default envelope 500–1500 words), `rubric.md`, consumer rubric overrides, `<thread>.{N}/_progress.json` (the drafter's self-check + `metadata.voice_exemplars`), and any previous review for this slug.
3. **Run the deterministic pre-flight gates (BEFORE any judgment)** and record every outcome in `_gate.json`:
   - **3a. Numeric consistency — BLOCKING** (issue #462's hook, built for this skill): invoke `uv run --project .anvil python -m anvil.lib.numeric_consistency <thread>.{N}/ --write-review --blocking`. The CLI writes the `<thread>.{N}.numeric/_review.json` sidecar via its own `staged_sidecar`; under `--blocking` it emits one `CriticalFlag` per finding-code cluster, which forces `Verdict.BLOCK` through `critics.compute_verdict`. Mirror each cluster as a **Numeric-consistency failure** critical-flag paragraph in `verdict.md` (rubric flag 7) quoting the claim, the named values, and the actual arithmetic. Per-claim suppression (`<!-- anvil-lint-disable: numeric_consistency -->`) is honored by the CLI (suppressed hits surface as info, never gate). An essay with zero numbers passes vacuously.
   - **3b. Hyperlink resolution — BLOCKING for broken/unresolvable links**: invoke `python -m anvil.lib.hyperlink_resolver <thread>.{N}/ --write-review` (the module promoted from the memo skill-local lib under this skill — #335/#460; add `--check-external` only when the operator asks for an online pass). The CLI writes `<thread>.{N}.hyperlinks/_review.json`. Unresolved cross-thread refs raise `critical_broken_cross_thread_anchor` (BLOCK via the aggregator); broken markdown-internal and wiki links arrive as `major` findings — **escalate them to a blocking verdict** when still unresolved at verdict time (step 8): a published essay with a dead internal link fails the publish-handoff guarantee. An essay with zero links passes vacuously (the coverage *judgment* at step 5d still runs).
   - **3c. Rhetoric lint — ADVISORY** (issue #463): run `anvil/lib/rhetoric_lint.py::lint_rhetoric` over the body. **Consumer rule resolution first** (issues #468/#479 — the `voice.rhetoric_rules` BRIEF sub-key, the memo-render step 4g contract ported to this direct call): call `anvil.lib.project_brief.resolve_rhetoric_rules(<project_dir>)` with the same `<project_dir>` (the BRIEF-bearing project root) that step 4 passes to `resolve_voice_docs` — derive it once and share it, don't re-derive. Then:
     - **`None` returned** (no BRIEF / malformed BRIEF / no `voice:` block / no `rhetoric_rules` sub-key / whitespace-only value) → **omit the `extra_rules_path=` kwarg** entirely; the framework default rule set (`anvil/lib/rhetoric_lint.py::DEFAULT_RHETORIC_RULES`) applies — defaults-only behavior is byte-identical whether or not a `voice:` block is present.
     - **Resolved entry with `missing == False`** → pass `extra_rules_path=entry.paths[0]` (project-root hit wins over consumer-root fallback; absolute declared paths bypass the walk — the `resolve_voice_docs` precedent).
     - **Resolved entry with `missing == True`** (declared-but-missing file) → **still pass the path**: `extra_rules_path=<project_dir>/<entry.declared>` (the declared path verbatim when it is absolute). Do NOT silently omit the kwarg — `lint_rhetoric`'s loader graceful-degrades to a defaults-only run **plus one warning finding naming the error**, so the broken declaration surfaces in the lint result and the existing `_gate.json` `rhetoric.warnings` count ("a defect to surface, not an opt-out"; zero new error machinery).

     Note the kwarg asymmetry with memo: memo-render forwards through `render_gate.gate(kind="memo", rhetoric_rules_path=...)`; this command calls `lint_rhetoric` **directly**, so the kwarg is `extra_rules_path=`. Consumer rules tune *which* findings fire, never their severity ceiling, and `rhetoric_rules` does not activate the voice-grounding judgment tier at step 4 (`resolve_voice_docs` never returns it; a `rhetoric_rules`-only `voice:` block keeps `VoiceDocs.is_empty` True). Findings are warning-ceiling **by contract — do NOT escalate severities, do NOT fold them into a critical flag, do NOT force `advance: false` from lint output alone**. They feed two judgment surfaces: deterministic evidence for the dim 9 justification (step 6) and assist material for the generic-AI-cadence flag (step 7), where the judgment call stays the reviewer's (irreducible false positives: quoted material, deliberate style).
   - **Write `_gate.json`** (inside the staging dir): `{ "numeric": {"ran": true, "blocking": true, "pass": <bool>, "findings": <count>, "critical_clusters": <count>}, "hyperlinks": {"ran": true, "pass": <bool>, "broken_internal": <count>, "broken_cross_thread": <count>, "external_probed": <bool>}, "rhetoric": {"ran": true, "advisory": true, "warnings": <count>, "infos": <count>} }`.
4. **Load voice grounding (conditional — issue #461)**: invoke `anvil/lib/project_brief.py::resolve_voice_docs(<project_dir>)` per `anvil/lib/snippets/voice_grounding.md`.
   - **When active**: read the resolved docs (values → style_guide → vocabulary → corpus exemplars); read `metadata.voice_exemplars` from the version's `_progress.json` to verify the drafter's grounding happened. Declared-but-missing files → a **`major` finding** in `comments.md` directing the operator to create or fix each file (tier stays active; record the missing paths for `_summary.md.voice_grounding.missing`). Cache the resolved list for steps 6 (dim 2), 7 (flags 1–2), and 9 (`_summary.md`).
   - **When inactive (no `voice:` block, `voice: {}`, or no BRIEF)**: record a **`major` finding recommending the operator declare the `voice:` block** — voice is the point of this artifact class; an absent contract is a defect to surface, not a crash and not silence (SKILL.md §Voice grounding — the deliberate divergence from memo's byte-identical-silence posture). Dim 2 still scores, uncalibrated; no `_summary.md.voice_grounding` block is emitted.
4b. **Load subject voice grounding (conditional — issue #598)**: invoke `anvil/lib/project_brief.py::resolve_subject_voice_docs(<project_dir>)` (the same `<project_dir>` as step 4; the **subject tier activates independently** of the author tier — a `subjects`-only `voice:` block returns `[]` from `resolve_voice_docs` but entries here) per `anvil/lib/snippets/voice_grounding.md` §"Subject voice tier".
   - **When active** (≥1 declared subject): read each subject's resolved `corpus` (spoken transcripts) + `voice_doc` (when present); read `metadata.subject_voice_exemplars` from the version's `_progress.json` to verify the drafter's per-speaker grounding happened. Declared-but-missing corpora/voice_docs → a **`major` finding** in `comments.md` per subject directing the operator to create or fix each file (tier stays active; record the missing paths for `_summary.md.subject_voice_grounding[].missing`). Cache the resolved subject list for step 6 (dim 2 per-subject pass), step 7 (Misattribution flag 8), and step 9 (`_summary.md`).
   - **When inactive** (no `subjects` list, empty list, or no BRIEF): the subject tier does not exist for this project — no finding, no `subject_voice_grounding` block (byte-identical to pre-#598). This is the customer-context activation convention, NOT the author-tier's declare-the-block recommendation: subjects are opt-in (most essays render no third-party dialogue), so their absence is silence, not a defect.
4c. **Provenance back-check (conditional — issue #611)**: invoke `anvil/lib/project_brief.py::resolve_corpus_dirs(<project_dir>)` (the same `<project_dir>` as steps 4 and 4b; the **corpus tier activates independently** of the voice tiers) per `anvil/lib/snippets/provenance.md` §Section 1.
   - **When active** (≥1 resolved dir): read `metadata.corpus_dirs_resolved` from the version's `_progress.json` to verify the drafter ran. **Spot-sample 5–10 rows** from `<thread>.{N}/provenance.md` per review pass, opening each cited `Source file` + `Line range` in the resolved corpus per `anvil/lib/snippets/provenance.md` §Section 3. Emit findings as **`kind: judgment`** with `evidence_span` pointing at the map row (`provenance.md:L<N>`), **quoting both the claim and the cited passage** (vague back-check feedback without a quoted passage is itself a defective finding, the §Section 3 evidence discipline). A row whose cited file is **not resolvable** under any corpus root → **`major`** finding. A row whose cited passage **does not support the claim as written** → **`blocker`** finding. A **missing `provenance.md`** when the tier is active → **`major`** finding directing the operator to re-draft with provenance tracking (a broken contract, not a crash). The back-check is a **sampling** check by design — the exhaustive five-way audit pass is out of essay's surface (essay ships no audit command; that is paper's follow-on).
   - **When inactive** (no `corpus:` key, `corpus: null`, or `corpus: []`): byte-identical to pre-#611 — no block, no findings, no `provenance_back_check` section in `_summary.md`.
   - **Declared-but-missing dirs**: the tier still activates; `resolve_corpus_dirs` returns `missing: true` entries (never raises) — surface the broken declaration as a **`major`** finding.
5. **LLM coherence passes (before scoring)** — the three sister moves ported from the consumer's blog-review, covering what craft scoring misses:
   - **5a. Dinner-party register lens** (step-2 port): does this read like *sharing at the dinner party*, or like *winning an argument*? Quote 1–2 lines that demonstrate the answer; apply the lens across every scored dimension; feeds dim 6 directly.
   - **5b. Example-coherence pass** (step-2.5 port — **BLOCKING via LLM critical flag**): restate the essay's central abstract claim in one sentence; restate the central worked example in one sentence; ask whether the example *physically needs* what the claim says is the gate. Also catch: metaphors fighting the underlying physics; cost claims conflating dollars/joules/latency/bandwidth/area; "X requires Y" where Y isn't binding; closing images trading plausibility for romance. On failure, raise the **Example-coherence failure** critical flag (rubric flag 6): quote the framing sentence AND the example, and state what the example actually needs (the toaster failure is the canonical instance). No detector exists by design (#462 gate 1) — this prose check is the gate.
   - **5c. Full claim-vs-claim numeric pass** (step-2.6 port): the step-3a detector is the deterministic extraction assist UNDER this pass, not a replacement for it. List every number; for each spread/gap/percentage/multiplier claim, do the arithmetic; flag claims referencing a broader population the essay never bridges to; catch the shapes the detector cannot prove (the same quantity stated twice disagreeing across sections, spelled-out numbers). Real failures raise the **Numeric-consistency failure** critical flag (rubric flag 7) with the arithmetic shown.
   - **5d. Link-coverage judgment** (step-2.7 port — `major`, never critical): for every specific named entity (paper, benchmark, leaderboard, project, organization, product), apply the dinner-party-in-reverse test — if a friend asked "where do I find that?", would the author hand over a URL? Unlinked references-worth-linking become `major` `comments.md` entries feeding dim 7. Skip household-name brands as background, common infrastructure, public figures (their *works* may still warrant links), and cultural-shorthand sampling. Missing links are polish, not blockers — this half never raises a critical flag (the deterministic broken-link half at 3b is the blocking surface).
6. **Score the 9 dimensions** per `rubric.md` into `scoring.md` (`# | Dimension | Weight | Score | Justification`, integer scores, 1–3 sentence justifications quoting evidence):
   - **Quoted-evidence requirement (issue #464 / #475)**: each dimension's justification MUST embed at least one **verbatim quote from `<thread>.md`**, wrapped in inline double quotes and followed by a location anchor — `("the quoted span" — §2.1)` — per `anvil/lib/snippets/rubric.md` §"Dimension scoring guidance" rule 1. Use inline `"..."` spans, NOT blockquotes (justifications live in single table cells). A dim scored at **full weight** MAY substitute the by-absence marker `no instance of <X> found` — absence of defects has no quotable span; below ceiling the quote requirement stands. The quote must be byte-verbatim from the body — a paraphrase presented in quote marks is fabricated evidence, the defect class the step 6b self-check exists to catch. **Elision with `...` / `…` is permitted** (issue #478): a quote may skip intervening text with an ellipsis, provided each elided fragment is itself verbatim, ≥ `MIN_QUOTE_CHARS` normalized chars, in document order, and drawn from one nearby passage (within the verifier's `ELISION_WINDOW_CHARS` proximity window) — do NOT stitch fragments from distant sections into one quote. Em/en dashes may be typed as `--` / `---` (the verifier folds dash variants symmetrically). (Voice deductions under the dim 2 sub-step quote BOTH the offending essay passage per this rule AND the corpus exemplar per `voice_grounding.md` — the two contracts are complementary; see the step 6b note on exemplar quotes.)
   - **Dim 2 (Voice fidelity — owned)**: when the voice tier is active, score against the resolved docs per `anvil/lib/snippets/voice_grounding.md` §Reviewer contract. **Every voice deduction MUST quote a corpus exemplar** showing what the target sounds like (count the quotes for `_summary.md.voice_grounding.exemplars_quoted`); apply the **convergence-with-Claude adversarial check** (*would I, the AI, also write this sentence?* — if yes, scrutinize harder, never defend); check voice signatures and vocabulary tells in prose (deterministic counting belongs to the 3c lint). Append the triggered suffix `"voice grounding active — dim 2 scored against <resolved values/style_guide paths>; voice deductions must quote corpus exemplars"` to the dim 2 justification (the #348 triggered-fixed-suffix precedent). When the tier is inactive, score dim 2 uncalibrated and note `"no voice contract declared — dim 2 scored without corpus calibration (see the major finding)"`.
     - **Per-subject voice-fidelity sub-pass (issue #598 — when the subject tier is active from step 4b)**: within this same dim 2 score, run a **per-subject pass** over each speaker's rendered dialogue against that speaker's resolved transcript corpus (+ `voice_doc` when present) per `voice_grounding.md` §"Subject voice tier". **Every subject-voice deduction MUST quote the transcript** showing the speaker's actual cadence **alongside the drifting reconstructed line** (quote the transcript, quote the drifting line) — vague feedback like `"doesn't sound like her"` without a transcript quote is itself a defective finding. Apply the **generalized convergence-with-Claude check**: *would I, the AI, also write this line for this speaker?* — if yes, scrutinize harder; a polished balanced sentence where the corpus shows clipped declaratives is the canonical failure mode. Per subject, count the transcript passages quoted (→ `subject_voice_grounding[].exemplars_quoted`) and the lines deducted (→ `subject_voice_grounding[].lines_flagged`). The per-subject deductions fold into the single dim 2 score — this is NOT a new rubric dimension and the rubric total stays /44. Append `"; subject voice tier active — <N> subject(s) scored against transcript corpora, subject-voice deductions must quote transcripts"` to the dim 2 suffix.
   - **Dim 9 (Rhetorical economy — load-bearing)**: cite specific anti-pattern instances (throat-clearing, trailing recaps, hedge cushions, non-load-bearing paragraphs, length past the envelope) AND cite relevant 3c lint findings as advisory evidence (never as forced deductions). **Echo rule**: every cited dim 9 instance ALSO appears as a `scope: reduce` entry in `comments.md` so the reviser sees the trim directive in the comment stream it consumes.
   - Remaining dims per their rubric.md rows; dim 6 consumes 5a; dim 7 consumes 5d plus the argument-chain judgment.
6b. **Validate quoted evidence (deterministic, write-time self-check)** — issue #464 / #475:
   - After the `scoring.md` write lands inside the staging dir, invoke `uv run --project .anvil python -m anvil.lib.evidence_check <thread>.{N}/ --scoring <staging dir>/scoring.md` (or call `anvil.lib.evidence_check::check_version_dir(<thread>.{N}/, scoring=<staging dir>/scoring.md)` directly). The verifier parses the scoring table via `anvil/lib/critics.py::parse_memo_scoring_table`, extracts the quoted spans from each justification, and checks each one against `<thread>.md` (curly→straight quote folding, dash-variant folding `—`/`–`/`---`/`--`, whitespace collapse, markdown-emphasis stripping; case-sensitive substring match, with `...`/`…`-elided spans matched fragment-by-fragment in document order within the `ELISION_WINDOW_CHARS` proximity window — issue #478). Classification per justification: ≥1 span matching the body → pass; score at full weight + `no instance of <X> found` marker → pass (ceiling-by-absence); spans present but none matching → **major `fabricated_evidence` finding**; no spans at all → minor `missing_evidence` advisory. Anchors are NOT validated (judgment-free scope). Dim 2 corpus-exemplar quotes (per `voice_grounding.md`) will NOT match the essay body — the verifier tolerates non-matching quotes so long as ≥1 quoted span in the same justification matches the body (classification rule 1, the calibration-suffix tolerance), so every dim 2 justification quotes the offending essay passage IN ADDITION to the exemplar.
   - **Findings are a write-time self-check failure — correct before the sidecar lands** (the memo-review step 7c posture): a `missing_evidence` finding means the reviewer adds the verbatim quote + anchor (or, at full weight, the by-absence marker) to that dimension's justification and re-runs the check. A `fabricated_evidence` finding is the hard case — the quoted span does not appear in the body, so the reviewer MUST re-derive that dimension's justification from the actual body text (re-read the section, re-quote verbatim, and reconsider whether the score itself was grounded) — exactly the lazy-critic failure mode the gate exists for. The check is deterministic and cheaply re-runnable; correction converges in one or two passes. The staged sidecar MUST NOT exit the context block while `fabricated_evidence` findings persist.
   - **Advisory boundary**: this self-check governs the reviewer's OWN staging-dir output only. It does NOT gate the verdict (no new critical-flag category, no change to the `advance` aggregation), does NOT write a sidecar, and is NEVER run retroactively against existing review dirs by this command — legacy review siblings are immutable and the rule applies to NEW reviews only.
7. **Identify critical flags** — the seven from `rubric.md`, each with a one-paragraph justification in `verdict.md` quoting the offending passage and the violated contract: anti-stance violation + out-of-standing claim (when the voice tier is active and the values doc declares stances/standing — routed through the existing critical-issue machinery per the #461 snippet, no new flag category), generic AI cadence (3c lint as assist; judgment here), factual error, unattributed borrowing, example coherence (5b), numeric consistency (3a + 5c). Plus, **conditionally, flag 8**:
   - **Misattribution — voice-identity failure (flag 8, conditional — issue #598)**: raised ONLY when the subject tier is active with **≥2 subjects declared**. When a line attributed to Subject A carries characteristic markers that match Subject B's corpus and **contradict** Subject A's corpus, raise this flag. This is the **voice-identity failure only** (wrong voice in the wrong mouth) — the substance-level "the underlying event belongs to another speaker's testimony" is out of scope here (→ #597), do NOT adjudicate it. The justification MUST cite: (1) the attributed line, (2) the Subject A corpus showing why it does not fit, and (3) when identifiable, the Subject B corpus showing why it does fit better. This is an **additive** flag routed through the existing critical-flag machinery (same `Verdict.BLOCK` consequence via `critics.compute_verdict`) — it does NOT change the rubric total or the advance threshold (the `_meta.json` stamps stay `rubric_total: 44` / `advance_threshold: 35`). With fewer than 2 subjects declared the flag cannot fire (a single speaker has no alternate corpus to misattribute against).
   - **Fabrication-class flags (flags 9–13, conditional — issue #611)**: raised ONLY when the corpus tier is active (i.e., step 4c's `resolve_corpus_dirs` returned ≥1 non-missing dir) AND the back-check spot-sample caught clear fabrication evidence. Uses the `CriticalFlag.type` vocabulary from `anvil/lib/snippets/provenance.md` §Section 6: `fabricated_quote` (verbatim-quoted text absent from the corpus), `fabricated_fact` (a named date/name/event not traceable to any corpus passage), `misattribution_of_substance` (an event or memory attributed to a speaker whose corpus does not contain it — the **substance-level** flag, distinct from flag 8's voice-identity failure), `anachronism` (an era-incompatible detail contradicted by the corpus chronology), `unattributed_paraphrase` (authorial invention presented as a subject's memory without any corpus grounding). Each flag's justification MUST quote the offending artifact text and the corpus evidence (or its explicit absence). These are **additive**: routed through the existing `Verdict.BLOCK` machinery (`critics.compute_verdict` — no change needed) and do NOT change the rubric total or advance threshold (the `_meta.json` stamps stay `rubric_total: 44` / `advance_threshold: 35`). With the corpus tier inactive, flags 9–13 cannot fire.

   If none: "Critical flags: none."
8. **Verdict** into `verdict.md`: total /44, critical-flag count, `advance: true` iff **total ≥35 AND zero unresolved critical flags AND the blocking gates pass** (3a clean or suppressed-only; 3b no broken internal/cross-thread links). Top 3 revision priorities: any gate failure first, then the highest-leverage dim deductions; when dim 9 scored below 4/5, at least one priority MUST be a `scope: reduce` item. List "What's working" — the moves the reviser must NOT sand off (voice signatures especially).
9. **Write `_summary.md`** (inside the staging dir): the rubric block `{ "id": "anvil-essay-v1", "total": 44, "advance_threshold": 35, "dimensions": 9 }` (+ `prior_rubric_id` when cached at step 1), the per-dim score map, the gates echo (mirroring `_gate.json`), `scope_distribution` `{preserve, expand, reduce}` counts over `comments.md`, and — **only when the author voice tier is active** — the `voice_grounding` block `{ran: true, docs_loaded: [...], exemplars_quoted: <n>}` (+ `missing: [...]` when declared docs were absent). When the author tier is inactive the block is NOT emitted at all (no `{ran: false}` entry — the customer-context activation convention per the #461 snippet; the recommendation lives in the `major` finding, not here).
   - **Subject voice grounding block (issue #598 — only when the subject tier is active from step 4b)**: emit a parallel top-level `subject_voice_grounding` block `{ran: true, subjects: [{name, corpus_files_loaded, voice_doc_loaded, exemplars_quoted, lines_flagged}, …]}` (+ `missing: [...]` on any subject entry whose corpus/voice_doc was declared-but-missing). `corpus_files_loaded` = resolved transcript paths read; `voice_doc_loaded` = whether a `voice_doc` resolved present + non-missing; `exemplars_quoted` / `lines_flagged` = the per-subject counts from the step-6 dim 2 sub-pass. When the subject tier is inactive the block is NOT emitted at all (no `{ran: false}` entry — byte-identical to pre-#598 output). The two blocks are independent: a subjects-only project emits `subject_voice_grounding` but NOT `voice_grounding`.
   - **Provenance back-check block (issue #611 — only when the corpus tier is active from step 4c)**: emit a parallel top-level `provenance_back_check` block `{ran: true, dirs_resolved: [...], rows_sampled: <n>, major_findings: <n>, blocker_findings: <n>}` (`dirs_resolved` = the resolved corpus dir paths; `rows_sampled` = the 5–10 rows opened; `major_findings` / `blocker_findings` = the back-check finding counts from step 4c). When the corpus tier is inactive the block is NOT emitted at all (no `{ran: false}` entry — the same no-`ran:false` convention as the voice-grounding block, byte-identical to pre-#611 output). This block is independent of the two voice blocks: a corpus-only project emits `provenance_back_check` but neither voice block.
10. **Finalize `_meta.json` + `_progress.json`** inside the staging dir (`_progress.json` LAST), then exit the `staged_sidecar` block — manifest verified, staging dir atomically renamed to `<thread>.{N}.review/`.
11. **Report**: e.g., `Reviewed the-loop-is-the-unit.1 → 31/44, 1 critical flag (numeric-consistency: the 70-point spread), gates: numeric FAIL / links PASS / rhetoric 3 warnings. Next: essay-revise the-loop-is-the-unit`.

## What essay-review does NOT do

- **Never edits the body.** Read-only against `<thread>.{N}/`.
- **Never escalates rhetoric-lint severities** (the #463 advisory contract is fixed) and never blocks on lint output alone.
- **Never blocks on missing links to named entities** (coverage is dim 7 `major` territory; only *broken* links block).
- **Never crashes on a missing voice contract** — the `major` finding is the surface (the `customer_context.py` posture).
- **Don't be vague** ("voice feels off" without a corpus quote is a defective finding — and for a subject line, "doesn't sound like her" without a *transcript* quote is equally defective, #598), **don't moralize**, **don't recommend balanced point-counterpoint structure**, **don't suggest cutting items on the "What's working" list**, **don't defend voice moves you would make yourself** (for subject dialogue: don't defend a line you, the AI, would also write for that speaker — that convergence is the drift signal).
- **Never adjudicate subject-line substance via the voice tier** — whether the underlying event/date/memory actually appears in the transcript corpus is the **corpus-provenance tier's** job (step 4c + flags 9–13, #611/#597), NOT the voice tier's. The Misattribution flag (8) is a *voice-identity* failure only; **substance-level** misattribution is `misattribution_of_substance` (flag 12) from the provenance tier. When the corpus tier is inactive, substance verification does not run at all — an ungrounded essay is scored on craft alone (byte-identical to pre-#611).

## Scorecard kind

This critic emits the `human-verdict` scorecard kind per `anvil/lib/snippets/scorecard_kind.md`. `_meta.json` MUST include `"scorecard_kind": "human-verdict"` plus the three rubric-stamping fields (`"rubric_id": "anvil-essay-v1"`, `"rubric_total": 44`, `"advance_threshold": 35`).

## Idempotence and resumability

Standard: a completed review is never re-run (sibling-exists check at step 1); a crashed review is re-runnable (the entry sweep removes this critic's stale staging dir; the gate CLIs sweep their own). Validation by file existence.

## Git sync (opt-in, off by default)

Per `anvil/lib/snippets/git_sync.md` (`.anvil/anvil/lib/snippets/git_sync.md` in an installed consumer repo): if `.anvil/config.json` exists and `git.commit_per_phase` is `true`, end this phase: stage only the dirs this phase wrote, commit as `anvil(<skill>/<phase>): <thread>.{N} [<state>]`, push if `git.push` is `true`. Git failures warn and continue — never fail the phase. When the config or knob is absent, skip this step entirely (default off).

This phase's specifics:

- **Ordering**: after the staged-sidecar atomic rename (issue #350) lands the final-named `<thread>.{N}.review/` — so only complete sidecars are ever committed.
- **Staging target**: ONLY this command's own outputs — `<thread>.{N}.review/` plus the `<thread>.{N}.numeric/` and `<thread>.{N}.hyperlinks/` gate sidecars its step-3 CLIs wrote (never other critics' dirs — the narrow scope keeps the hook safe under parallel fan-out).
- **Commit**: `anvil(essay/review): <thread>.{N} [REVIEWED]`.
