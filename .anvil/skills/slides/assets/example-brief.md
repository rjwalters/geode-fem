# Talk brief: Agentic AI in Production — What Engineers Should Know

## What this talk is

A 30-minute internal talk for software engineers introducing the design space of agentic AI workflows. Goal is to give the audience enough vocabulary and pattern recognition to engage substantively in design reviews where an agent-based feature is proposed — including the right questions to ask about reliability, observability, and cost.

## What this talk is not

- A coding workshop (no live coding; no framework recommendations).
- An LLM tutorial (assumes the audience already uses LLMs as tools).
- A vendor comparison (no recommendations on OpenAI vs. Anthropic vs. Google).

## Key references

- The talk speaker has 18 months of experience operating production agentic workflows.
- Anecdotes drawn from internal experience; no external citations required for this audience.
- Pattern names borrowed from the public literature where convenient (e.g., "tool use", "self-reflection", "human-in-the-loop").

## Speaker notes

- Audience size ~80; room is medium-sized, projection screen visible from all rows.
- Q&A is encouraged throughout (not held to the end).
- The talk is being recorded for engineers who couldn't attend in person.
