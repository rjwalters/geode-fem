# primer-figures — Figurer

**Role**: figurer (renders the drafter-planned teaching diagrams + an optional PDF).
**Reads**: latest `<thread>.{N}/<thread>.md` + `_progress.json` (specifically `metadata.figure_plan`), `<thread>/refs/` (diagram source when the author supplies mermaid `.mmd` or figure specs), project `BRIEF.md`.
**Writes**: `<thread>.{N}/exhibits/` (rendered figures at exactly the drafter-specified paths) and, optionally, `<thread>.{N}/<thread>.pdf` — all inside the version dir's exhibits/render slots; the markdown body is never edited.

`primer-figures` **fills in the exact `exhibits/…png` paths the body already references** (the drafter placed those references + recorded the figure plan per `primer-draft.md` step 5b — the `report-figures` "scan the body for exhibit references, render to those paths" precedent). It is **collateral, not a state advance** — it does not move the state machine. Post-#690 it runs **any time after draft/revise** (not gated on `AUDITED`), mirroring `report-figures`, so that `primer-review`/`primer-audit` can see and score the rendered figures (dim 3 worked-example concreteness, dim 7 structure). It MAY also be re-run post-`AUDITED` as an idempotent refresh (e.g., after a `spec_ref` change), but the primary invocation is mid-lifecycle.

## Output format (reuses the shared render pipeline — no new plumbing)

Follows `report`'s markdown-source-of-truth + optional-PDF precedent exactly (SKILL.md §Output format):

- **Diagrams via `mmdc → PNG`** — the documented working diagram path (`report`/`paper` figure primitives). Message flows, commitment/coin lifecycles, and the end-to-end walkthrough are authored as mermaid and rendered to PNG. Inline mermaid leaks as raw code in PDF (WORK_LOG PR #72) — the diagrams are rendered to PNG and the body **already references them** (`![Figure N — caption](exhibits/figN-slug.png)`, placed by the drafter per `primer-draft.md` step 5b); the mermaid source is never embedded in the body. The figurer's job is to make each referenced path resolve to a real file — it does not invent new references (that is a body edit).
- **Caption-numbering convention (the drafter/figurer agreement, #690)**: captions carry their own `Figure N —` prefix in the body reference's alt text. The render defaults set `\usepackage{caption}\captionsetup{labelformat=empty}` so pandoc's implicit-figure path does NOT *also* prepend a "Figure N:" label — without this the render shows the doubled "Figure N: Figure N — …" reported from the botho#881 canary run. Bare, unnumbered captions are NOT the convention: the author numbers, the renderer stays out of numbering. This directive lives in the single `include-in-header` block of `anvil/skills/primer/assets/pandoc-defaults.yaml` (see below) — it is the one source of truth, co-located with the `\pandocbounded` height cap so the two directives cannot clobber each other.
- **PDF via pandoc (primary) / LaTeX (opt-in)** — `<thread>.pdf` is produced from `<thread>.md` via `anvil/lib/render.py` using the pandoc path by default. The primer pins the **xelatex/LaTeX pandoc path** via `anvil/skills/primer/assets/pandoc-defaults.yaml` (`--defaults <that file>`) — invoke `pandoc <thread>.md -o <thread>.pdf --defaults anvil/skills/primer/assets/pandoc-defaults.yaml` (consumers override via `.anvil/skills/primer/assets/pandoc-defaults.yaml`). A consumer needing precise typography can additionally drop a `.tex` template under `.anvil/skills/primer/assets/` for the fuller LaTeX opt-in path (mirroring `report`'s "primary: pandoc, secondary: opt-in LaTeX"). **No third rendering path is invented.** The defaults asset carries, in its one `include-in-header` block: (a) the `\captionsetup{labelformat=empty}` caption directive (#690/#695); and (b) a `\pandocbounded` override capping images at `0.85\textheight` (#692) — pandoc ≥3.10's default `\pandocbounded` fills `\linewidth × \textheight` with no headroom for a caption, so a tall figure + multi-line caption overflows the page (captions overlapped the page footer / clipped at the edge in the canary). The cap reserves room for the caption on the same page.

The version dir is self-contained for archival: `<thread>.md` (source-of-truth, containing the figure references), `exhibits/` (rendered figures at the referenced paths), `<thread>.pdf` (optional render) side by side.

## Procedure

1. **Discover state**: find the highest `N` with `<thread>.{N}/<thread>.md` (slug-echo per #295). **No terminal-state gate** — `primer-figures` runs any time after the draft exists (post-#690), mirroring `report-figures`. It needs a drafted body with a figure plan, not a converged one; running it mid-lifecycle is the intended path so review/audit can score the rendered output. (A version that is already `AUDITED` is also fine — the phase is idempotent.)
2. **Resume check + resolve the figure plan**: read `<thread>.{N}/_progress.json.metadata.figure_plan` AND scan `<thread>.md` for `![…](exhibits/<filename>)` references — the two must agree (the plan is the drafter's record; the body references are the source of truth for what to render). The render targets are **exactly the `exhibits/<…>.png` paths the body references** — never invent a new path. If every referenced exhibit already exists AND `<thread>.pdf` is newer than `<thread>.md` AND `phases.figures.state == done`, exit early (idempotent, the `report-figures` resume rule). **Zero-figure thread** (`figure_plan` empty/absent AND no `![…](exhibits/…)` references in the body): rendering diagrams is a silent no-op; proceed to the optional PDF (step 4) directly — byte-identical to a diagram-less run.
3. **Render diagrams to the referenced paths**: for each entry in the figure plan / each body reference, render its mermaid `source` (a `.mmd` under `<thread>/refs/` or the drafter's recorded inline spec) to PNG via `mmdc` and land the output at **exactly the `path` the body references** (`exhibits/figN-slug.png`). A missing `mmdc` binary degrades gracefully (per the `check_*_available()` family in `anvil/lib/render.py`): skip the diagram, record the gap in `_progress.json.metadata.figures.skipped`, never abort the phase (the broken body reference then remains, and the review's step-4c existence check will surface it as a dim-3/7 finding). **`mmdc` launchability, not just presence (#692)**: `check_mmdc_available()` only tests that the binary is on PATH — it can report available while `mmdc`'s pinned Puppeteer/Chromium is absent from `~/.cache/puppeteer`, so every diagram render fails at browser launch. Before committing to a batch of diagram renders, call `check_mmdc_launchable()` (a trivial probe render). On `False`, surface `MMDC_LAUNCH_REMEDIATION` and record the gap in `metadata.figures.skipped` (same graceful-degrade path as an absent binary — the remediation is either `npx puppeteer browsers install chrome` or a puppeteer-config `executablePath` override pointing mmdc at a system Chrome).
4. **Render-gate pre-flight (deterministic)**: run `anvil/lib/render_gate.py` over the render inputs (placeholder scan, compile-success check) before the expensive PDF render — the framework-wide "deterministic pre-flight before judgment" pattern. The caption defaults (`\captionsetup{labelformat=empty}`, §Output format) are applied on this render so figure captions are not double-numbered. **Post-render assertions (#692)**, run over the produced PDF via `gate(pdf_path=…, source_paths=[<thread>.md], …)` (both are error-severity — they block the gate, not merely warn):
   - **Glyph verification** (`glyph_verification` dim): the gate sweeps the body for **every** non-ASCII codepoint and asserts each survives into the `pdftotext` extraction at ≥ its source count. This is source-driven, not allow-list-driven — it catches the STIX Two Text `≠` (U+2260) silent glyph-drop the canary shipped (a hardcoded known-glyph list missed it by construction). Degrades gracefully with a `reasons` breadcrumb when `pdftotext` (poppler-utils) is absent.
   - **Embedded-image assertion** (`embedded_images` dim): the gate counts `![…](path)` references in the body and asserts `pdfimages -list` reports ≥ that many embedded images. This catches the botho v2 "zero embedded images, every other gate green" failure deterministically. Degrades gracefully when `pdfimages` (poppler-utils) is absent.
5. **Produce the optional PDF**: invoke `anvil/lib/render.py` to render `<thread>.md` → `<thread>.{N}/<thread>.pdf` via the pandoc path (or the LaTeX opt-in when a consumer `.tex` template is present). A missing `pandoc` binary degrades gracefully — record the gap, do not abort.
6. **Validate by file existence (mirrors `report-figures.md` "Validation by file existence")**: after rendering, assert that **every `![…](exhibits/<filename>)` reference in `<thread>.md` resolves to a file that now exists** under `<thread>.{N}/exhibits/`. Any reference whose target is still missing (e.g., `mmdc` was unavailable) is recorded in `metadata.figures.unresolved` — this is the deterministic record `primer-review`'s dim-3/7 existence check reads (it is NOT a fatal error here; graceful degradation, surfaced at review). This is the "broken refs before the figures phase are fine, but after it they should resolve" contract from #690.
7. **Record provenance** into `<thread>.{N}/_progress.json`: `phases.figures.state = done` (LAST write), `metadata.figures.rendered` (the PNGs produced, keyed to their figure-plan ids), `metadata.figures.pdf` (the PDF path or `null` when the renderer was unavailable), `metadata.figures.skipped` (renderer-unavailable gaps), `metadata.figures.unresolved` (referenced-but-missing exhibit paths after this run).
8. **Report**: e.g., `Figured botho-from-the-basics.2 → 5 diagrams under exhibits/ (all referenced paths resolved), botho-from-the-basics.pdf rendered (pandoc). Next: primer-review + primer-audit can now score the rendered figures.`

## What primer-figures does NOT do

- **Never invents new body content or new figure references.** The body — written by the drafter/reviser — already contains the `![Figure N — caption](exhibits/…)` references; the figurer only fills in the files those references point at. It renders to the drafter-specified paths; it does not add, move, or reword references (those are draft/revise's job). This is the narrowed successor to the old "never edits the markdown body" rule: the body is still source-of-truth and the figurer still never writes to `<thread>.md` — the point is only that the references it's filling in were placed upstream, so there is no contradiction with the "reference the image" guidance any more (#690).
- **Never invents a rendering pipeline** — reuses `anvil/lib/render.py` + `anvil/lib/render_gate.py`. The consumer-pluggable figure-adapter registry (`report.figure_adapters`) and the LaTeX/TikZ authoring path are deferred (SKILL.md §Deferred).
- **Never advances the state machine** — figures are collateral, not a state advance; the phase is idempotent and may run mid-lifecycle (post-#690) or after `AUDITED`.
- **Never aborts on a missing renderer binary** — graceful degradation (the `check_*_available()` precedent); the gap is recorded (`metadata.figures.skipped` / `.unresolved`) and surfaced at review, not fatal.

## Git sync (opt-in, off by default)

Per `anvil/lib/snippets/git_sync.md`: if `.anvil/config.json` exists and `git.commit_per_phase` is `true`, end this phase: stage only the dirs this phase wrote, commit as `anvil(<skill>/<phase>): <thread>.{N} [<state>]`, push if `git.push` is `true`. Git failures warn and continue. Default off.

This phase's specifics:

- **Ordering**: after the `_progress.json` `done` write lands.
- **Staging target**: ONLY this command's own `<thread>.{N}/exhibits/` + `<thread>.{N}/<thread>.pdf`.
- **Commit**: `anvil(primer/figures): <thread>.{N} [<state>]` (the bracket carries the thread's current derived state per SKILL.md §State machine — the figures phase does not advance the state machine, and post-#690 it may run before the thread is `AUDITED`).
